# 문제 정의 및 요구 사항


# 🎙️ 콘서트 예약 서비스

## 문제 정의

**콘서트 예약 서비스**는 유저가 대기열을 통과한 후, 좌석을 예약하고 결제하는 기능을 제공한다.

이 과정에서 **대기열 관리, 동시성 제어, 예약 시스템**을 고려해야 한다.

### **해결해야 할 주요 문제**

1. **대기열 관리**
    - 유저는 대기열을 통과해야만 예약이 가능해야 한다.
    - 유저에게 **대기열 토큰**을 발급하고, 이를 기반으로 API 접근을 제한해야 한다.
2. **좌석 예약 & 결제**
    - 예약 요청이 들어오면 **임시 배정(10분)** 상태로 좌석을 유지해야 한다.
    - 결제가 완료되지 않으면 **자동으로 예약 취소**해야 한다.
    - 예약 목록에 상태를 결제 취소로 설정해야 한다.
    - **동시성 제어**를 통해 동일 좌석이 중복 예약되지 않도록 해야 한다.
3. **잔액 충전 & 관리**
    - 사용자는 사전에 잔액을 충전하고 이를 사용해 좌석을 예약해야 한다.
    - 결제 시 잔액이 부족하면 예약이 실패해야 한다.

## 요구사항 분석

### 기능적 요구 사항

- 잔액 충전
- 잔액 조회
- 콘서트 목록 조회
- 콘서트 회차 조회
- 콘서트 좌석 조회
- 콘서트 좌석 예약
- 예약 좌석 조회
- 예약 취소
- 예약 좌석 결제
- 예약 좌석 환불
- 대기열 토큰 발급
- 대기열 조회

### 비 기능적 요구 사항

**성능 (Performance)**

**대기열 처리 속도**

- 대기열이 많아지더라도 빠르게 처리 가능해야 한다.
- 여러 사용자가 동시에 같은 좌석을 예약하는 상황을 방지해야 한다.
- 예약 후 일정 시간이 지나면 **자동으로 해제**되도록 스케줄링 필요하다.

**보안 (Security)**

- 유저의 결제 정보는 반드시 암호화 저장 및 전송
- **HTTPS 적용 & 민감 정보 보호**
